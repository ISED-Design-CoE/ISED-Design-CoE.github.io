Notes about this mockup:

  • If the user has saved their progress then we already have an email address. Populate it into the email field, but let them change it
  • This page moves the email field first, per https://uxmovement.com/forms/why-the-email-confirmation-field-must-die/
  • It would be a good idea to add some sanity checks as well. Some ideas:
    • Check whether the domain (the part after the @ sign, if it's the kind of email address that includes an @ sign) looks like a typo for one of the widely-used email providers
      • This list would include domains like gmail.com, yahoo.com and yahoo.ca, hotmail.com and hotmail.ca, outlook.com, and aol.com
      • We could also look at the domains of email addresses already in our database
    • If it does, ask the user whether they meant that address instead (example dismissable alert: "Did you mean yahoo.com? Change or dismiss")
    • Otherwise, check whether an MX record exists for the domain, and give the user a warning message (e.g. "Please ensure email address is typed correctly") if it doesn't
  • Error message if the user clicks Next without filling all the fields: "Please fill in all fields"
